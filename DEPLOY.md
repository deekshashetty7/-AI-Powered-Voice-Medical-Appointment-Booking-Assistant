# MediVoice Deployment Guide

Production stack:

| Service | Host | Directory |
|---------|------|-----------|
| Frontend | **Vercel** | `frontend/` |
| Backend API | **Render** or **Railway** | `backend/` |
| Voice agent | **Render worker** or **Railway** | `agent-node/` |
| Database | **Neon PostgreSQL** | external |

---

## 1. Database (Neon)

1. Create a project at [neon.tech](https://neon.tech)
2. Copy the **pooled** connection string → `DATABASE_URL`
3. After first backend deploy, run seed once (see step 3)

---

## 2. LiveKit Cloud

1. Create a project at [cloud.livekit.io](https://cloud.livekit.io)
2. Copy `LIVEKIT_URL`, `LIVEKIT_API_KEY`, `LIVEKIT_API_SECRET`
3. Use the same credentials on **backend** and **agent-node**

---

## 3. Backend (Render or Railway)

### Render (recommended)

1. Connect your GitHub repo
2. **New Blueprint** → use `render.yaml` at repo root
3. Set these secrets when prompted:

| Variable | Example |
|----------|---------|
| `DATABASE_URL` | `postgresql://...neon.tech/neondb?sslmode=require` |
| `LIVEKIT_URL` | `wss://your-app.livekit.cloud` |
| `LIVEKIT_API_KEY` | from LiveKit dashboard |
| `LIVEKIT_API_SECRET` | from LiveKit dashboard |
| `JWT_SECRET` | long random string (auto-generated by Render) |
| `ADMIN_INVITE_CODE` | `trikon-admin-2026` |
| `FRONTEND_URL` | `https://your-app.vercel.app` |

4. After first deploy, seed the database (one time):

```bash
# From your machine with DATABASE_URL set
cd backend && npm run db:seed
```

5. Note your backend URL, e.g. `https://medivoice-backend.onrender.com`

If you deploy the backend manually in the Render dashboard instead of using the blueprint:

- **Root Directory:** `backend`
- **Build Command:** `NPM_CONFIG_PRODUCTION=false npm ci && npm run db:generate && npm run build`
- **Start Command:** `npm run db:push && npm start`
- **Health Check Path:** `/health`

### Railway

1. New project → **Deploy from GitHub** → select `backend/` as root
2. Add the same environment variables as above
3. Railway reads `backend/railway.toml` for build/start commands
4. Run `npm run db:seed` once after first deploy

### Health check

`GET /health` → `{ "status": "ok" }`

---

## 4. Voice Agent (agent-node)

> Use **agent-node** (Node.js), not the Python `agent/` folder, for production.

### Render

`render.yaml` deploys `agent-node` as a **worker** service automatically.

Set these env vars on the worker:

| Variable | Value |
|----------|-------|
| `LIVEKIT_URL` | same as backend |
| `LIVEKIT_API_KEY` | same as backend |
| `LIVEKIT_API_SECRET` | same as backend |
| `GROQ_API_KEY` | from [console.groq.com](https://console.groq.com) |
| `DEEPGRAM_API_KEY` | STT provider |
| `SARVAM_API_KEY` | optional |
| `ELEVENLABS_API_KEY` | optional |
| `BACKEND_URL` | auto-linked from backend service (Render adds `https://`) |

### Railway

1. New service → root directory `agent-node/`
2. Uses `agent-node/Dockerfile` via `railway.toml`
3. Set the same env vars as above
4. Set `BACKEND_URL` to your backend's public URL (with `https://`)

### Verify agent

In LiveKit Cloud dashboard → **Agents**, confirm `medivoice-agent` is registered and idle/running.

---

## 5. Frontend (Vercel)

**Option A — Root Directory: `frontend` (recommended)**

1. Import GitHub repo at [vercel.com](https://vercel.com)
2. **Root Directory:** `frontend`
3. Framework: **Vite**
4. Environment variable:

| Variable | Value |
|----------|-------|
| `VITE_API_URL` | `https://your-backend.onrender.com` (no trailing slash) |

5. Deploy

**Option B — Root Directory: empty (repo root)**

Root `vercel.json` builds **only the frontend** — it does not run the backend `tsc` build.

Same `VITE_API_URL` env var required.

### CORS

Backend `FRONTEND_URL` must match your Vercel URL exactly:

```
FRONTEND_URL=https://your-app.vercel.app
```

For preview deployments, add comma-separated URLs:

```
FRONTEND_URL=https://your-app.vercel.app,https://your-app-*.vercel.app
```

## 6. Post-deploy checklist

- [ ] `GET https://<backend>/health` returns OK
- [ ] `GET https://<backend>/api/clinic` returns clinic data
- [ ] Frontend loads at Vercel URL
- [ ] Patient login/signup works
- [ ] Admin login (`admin@gmail.com`) routes to dashboard
- [ ] Voice call connects (backend + agent + LiveKit all running)
- [ ] Rotate any API keys that were shared in development

## Local development

```bash
# Terminal 1 — backend
cd backend && npm run dev

# Terminal 2 — voice agent
cd agent-node && cp .env.example .env  # fill keys
npm run dev

# Terminal 3 — frontend
cd frontend && npm run dev
```

Leave `VITE_API_URL` empty locally — Vite proxies `/api` to port 3001.

## Troubleshooting

| Issue | Fix |
|-------|-----|
| CORS error on login | Set `FRONTEND_URL` on backend to exact Vercel URL |
| Voice connection fails | Ensure agent worker is running; check LiveKit agent dispatch name `medivoice-agent` |
| 401 on admin stats | Sign in with admin account; check `JWT_SECRET` is set |
| Agent can't book | Set `BACKEND_URL` on agent to backend's `https://` URL |
| DB empty | Run `cd backend && npm run db:seed` once |
